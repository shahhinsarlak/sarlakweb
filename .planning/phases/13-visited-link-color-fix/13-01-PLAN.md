  <action>
Add the following CSS rule to `app/globals.css`, immediately after the `body { ... }` block (after line 30, before the `h1, h2, ...` heading rule) so it sits near other base element resets:

```css
a, a:visited {
  color: inherit;
}
```

Using `color: inherit` rather than a specific color value ensures every link inherits whatever color its parent or class assigns — whether that is `var(--text-color)`, `var(--accent-color)`, or any other value. The `a` selector without `:visited` is included to ensure the base anchor color also inherits by default, matching the existing `.link` and `.nav-link` class behavior for any unstyled anchors.

Do NOT add `text-decoration: none` here — that would remove underlines from links that intentionally have them. Only override color.

Do NOT add this rule inside any media query or scoped selector — it must be a top-level global rule.
